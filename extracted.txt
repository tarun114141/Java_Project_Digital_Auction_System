Relationship Summary   
• One USER (ADMIN) conducts many AUCTION_EVENT   
• One AUCTION_EVENT contains many ITEM   
• One USER (SELLER) lists many ITEM   
One CATEGORY groups many ITEM   
• One ITEM receives many BID   
• One USER (BUYER) places many BID   
• One ITEM has one PAYMENT (after sold)   
• One USER (BUYER) makes many PAYMENT
